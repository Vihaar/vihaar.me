## Site Structure

#### 1. Home – Life Map
- **Route**: `/`

  - Single big illustrated map (`life-map.png`) centered on the page.

    - Ski slopes / mountains.

    - Title (e.g., “Boxing”).
    - Subtitle.

#### 2. Family – Pass the Love
- **Route**: `/family`
- **Theme**: Warm reds, oranges, and dark brown background; family is the heart of everything.
- **Story beats**:
  - Michigan suburbs, growing up on a lake.
  - Home as a feeling more than a location.

  - Simple top-down view of family members (“Mom”, “Dad”, “Vihaar”, “Sibling”).

## Interaction System

- Hover → element lifts slightly and feels tactile
- Click → smooth zoom-in or camera movement into that world; avoid abrupt route changes
- Cursor → custom cursor or hand-drawn hover indicator is encouraged
- Transitions → always animated; no instant jumps unless stylistically necessary
- Feedback → every interaction should produce visible response
- Optional sound design → subtle heartbeat, paper rustle, wind, snow, crowd, kitchen sounds, etc.

### Specific Motion Rules
- Hovered map regions should elevate slightly and reveal subtle light/shadow under them
- The center heart should pulse continuously and feel alive
- Water should shimmer or move subtly
- Background elements can drift slowly (snow, fog, particles, paper, light beams)
- Animations should feel organic and hand-crafted, not robotic

---

## Map Behavior

- The map is NOT static
- It should feel alive and subtly animated at all times
- Paths should feel hand-drawn
- Regions should visually blend into each other rather than feel like separated boxes
- The map should function as the main hub into all life chapters
- Entering a chapter should feel like traveling into a story world, not opening a normal page

---

## Game Implementation

- All mini-games should be implemented using HTML5 Canvas or lightweight JavaScript game loops
- Do NOT use heavy game engines
- Mechanics should be simple, intuitive, and polished
- Each mini-game should:
  - Start quickly
  - Be replayable
  - Have a score, timer, round, or progress state
  - Have an obvious win/lose or completion condition
  - Match the emotional energy of the chapter it belongs to

---

## Visual Style Rules

- All visuals should feel hand-drawn, illustrated, or storybook-like
- Imperfection is GOOD: uneven lines, organic shapes, slightly rough forms
- Avoid sharp corporate rectangles; prefer soft, irregular, childlike forms
- Text should feel embedded in the world, like labels in a map or a storybook page
- The whole site should feel like it came from the inner child of Vihaar
- The design should be playful and childish, but still emotionally beautiful and cinematic

### Color System
- Base background: warm cream / paper tone
- Core palette: earthy oranges, reds, moss greens, muted blues, soft browns
- Accent colors can vary per chapter
- Avoid neon, glassmorphism, glossy gradients, or futuristic SaaS aesthetics

### Style Inspiration
- Studio Ghibli environment art
- Children’s illustrated storybooks
- Hand-drawn fantasy maps
- Warm cinematic adventure posters
- American Indian–inspired geometric prints and border motifs used thoughtfully and sparingly

- This should be built as a full working website, not a mockup
- All routes/pages should be implemented
- All navigation should be wired
- Every mini-game should be implemented, even if v1 is simple
